1. 회원가입
	회원 가입 양식
		- 유효성 검사
			필수 데이터 체크 (회원아이디, 회원명, 비밀번호 ....)
			아이디 유효성 검사
			비밀번호 유효성 검사 -> 복잡성
		- 비밀번호 처리(HASH)
			해시	-> 단방향 암호화 (복호화 불가)
					-> 원데이터를 알수 없음 -> 탈취해도 쓸수 없는데이터
					
			고정해시 - md5, sha1, sha256, sha512
				sha512
				
				같은 값 -> 같은 해시 값(해시 값의 일치 여부)
				
			유동해시
				같은 값	-> 해시를 생성할때마다 해시 값이 변경
							-> 예측 힘듬 (강한 보안)
				
				bcrypt
					- password_hash = PHP
					- bcrypt.hash = Node.js
				
				검증 -> 같은 값 != 같은 해시
					- password_verify = PHP
					- bcrypt.compare = Node.js
	
	DB에 저장 처리
	
2. 로그인
	회원이 아이디, 비번 입력
		- 유효성 검사(아이디, 비밀번호를 입력 했는지 여부)
		- 아이디로 회원정보를 추출 -> 회원정보 유무를 체크
		- 회원 정보가 있는경우	-> 비밀번호 검증(password_verify, bcrypt.compare)
										-> 비밀번호 일치, 불일치 체크
		- 비밀번호가일치	-> 세션에 회원 데이터를 조회 할 수 있는 단서 데이터를 저장
			세션, 쿠키 -> 개인화된 데이터
			쿠키 = 브라우저에 저장
			세션 = 서버에 저장(쿠키가 반드시 필요 - 서버에서 세션을 찾기위한 키값 - 쿠키로 저장)
			
			회원 단서 데이터 -> 세션에 저장하는 이유
